Summary App Usability Reflection

Usability Evaluation

1. Usability Test Plan

This evaluation assesses the effectiveness, learnability, and user satisfaction of Gratitude Garden, an Android app focused on calm and reflective gratitude journaling.

Goals

  • Verify that users can complete core workflows successfully.
  • Identify potential learnability issues for first-time users.
  • Measure perceived usability using the System Usability Scale (SUS).
  • Identify usability issues using Nielsen’s heuristics.

Approach

  • Heuristic evaluation of the initial prototype.
  • Moderated user test with task-based protocol and think-aloud method.
  • Post-test SUS questionnaire.

2. Heuristic Evaluation (Initial Prototype)

A heuristic evaluation was conducted on the initial prototype using common usability heuristics discussed in EMH-UEE, including visibility of system status, consistency, error prevention, and user control.

Strengths

  • Clear visual hierarchy and consistent styling across screens.
  • Navigation structure (Garden, Add Entry, History) is easy to understand.
  • Visual feedback (colors, highlights, calendar dots) supports orientation.

Identified Usability Issues

  • Date selection via calendar or month dropdown is not immediately obvious.
  • Icons without text labels rely on prior user familiarity.
  • The calendar visualization may require explanation for first-time users.

These findings informed the design of the user test tasks and research questions.

3. Hypotheses & Testable Questions

  • H1: Users can navigate between Garden, History, and Add Entry without external help.
  • H2: Users understand how to add a new gratitude entry, including selecting date and mood.
  • H3: Users can correctly interpret the calendar’s visual indicators.
  • Q1: Do users understand the meaning of colored calendar days and rings?
  • Q2: Where do users feel uncertain or confused?

4. Planned Data to Collect

Quantitative

  • Task completion rate.
  • Time on task (per task).
  • Number of errors or incorrect interactions.
  • SUS questionnaire scores.

Qualitative

  • User comments during and after tasks.
  • Observed hesitation or confusion.
  • Open-ended feedback on design and interaction.

5. Methods Used

The usability test was conducted as a moderated, in-person test using a think-aloud protocol. All participants completed the same set of tasks (within-subject design).

Participants

The test was conducted with 5 participants, aged approximately 20–35. All participants were regular smartphone users and had no prior experience with the app.

Tasks

  • Navigate to the Garden screen and explain what is visible.
  • Add a new gratitude entry for today.
  • Change the date of an entry.
  • Navigate to History and find an entry from another month.
  • Open an existing entry and describe its content.

6. User Test Results

Overall usability was rated very highly. SUS scores ranged from 82.5 to 100, indicating excellent perceived usability. Tasks were completed quickly with very few errors.

7. User Feedback & Observations

What users liked

  • Calm, muted color palette.
  • Clean and non-overwhelming design.
  • Clear visual structure and pleasant aesthetics.

Pain points

  • Uncertainty around mood selection and highlighting.
  • Some entries appeared editable when they were not.
  • Confusion about entry ordering.

8. Summary

The usability evaluation indicates excellent user satisfaction and strong learnability. Identified issues are minor and mostly related to visual affordances rather than structural problems. Overall, the app provides a calm, intuitive, and pleasant user experience.
