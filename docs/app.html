App Documentation

Concept & Idea

Gratitude Garden explores gratitude journaling through a calm visual metaphor. Instead of tracking streaks or statistics, users nurture a single plant that grows as entries are added over time.

Target Users

The app is designed for young adults who want to reflect regularly without feeling pressure, competition, or obligation.

App Structure

  • Garden (Home Screen)
  • Add Entry
  • History / Calendar
  • Entry Detail & Edit
  • Settings

User Flow

The following screenshots represent the final high-fidelity implementation of Gratitude Garden and reflect the intended user flow of the application.

The core user flow starts on the Garden (Home) screen, which visualizes the user’s overall gratitude progress through the growing plant. From there, users can:

1. Navigate to Add Entry to create a new gratitude entry by writing a short text, selecting a mood, and choosing a date.
2. Use the History / Calendar view to browse past entries by month and day.
3. Open an Entry Detail screen to read and edit an existing entry.
4. Access Settings for basic app configuration.

The navigation structure is intentionally simple and consistent to support a calm, low-pressure user experience and to minimize cognitive load for first-time users.

SQLite Database Design (ERD)

Entity Attribute Type Description
GratitudeEntry id Integer (PK) Unique identifier for each entry
text String Gratitude text written by the user
mood String Selected mood (e.g. Happy, Calm, Grateful)
timestamp Long Date of the entry stored as Unix timestamp

The database uses a single entity to keep the data model simple and fully supports offline-first CRUD operations.
